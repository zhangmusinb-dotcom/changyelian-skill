# 抖音财经内容合规规则

本文件用于约束“被套但体面”的产业链科普内容。目标是降低违规、限流和误导风险，但不承诺一定不限流、一定进精选或一定签独家。

## 核心原则

1. 内容必须是公开资料学习和产业链科普。
2. 行业是主线，公司只是案例。
3. 不提供任何买卖建议。
4. 不预测涨跌。
5. 不引导私信、进群、评论区发代码。
6. 不把免责声明当成掩盖实质荐股的手段。
7. 默认口播稿不出现具体上市公司名称、股票代码、简称、排名或并列名单；内部研究可以查公司和公告，但成稿要改写成产业角色。

## 绝对禁止

- 荐股
- 股票池
- 买卖建议
- 目标价
- 预测涨跌
- 承诺收益
- 私信引流
- 评论区发代码
- 收费带单
- 受益标的名单
- 低位潜伏
- 龙头股推荐
- 暗示操作
- 具体上市公司名称连续出现
- 股票代码或交易所简称
- “某某股份/某某科技/某某电路”等可直接映射到上市公司的名称
- “A 股/港股/美股某公司”这类投资语境
- “财报暴露”“订单兑现”“毛利率改善”与具体公司名称绑定
- “老师带你赚钱”
- “下周重点看”
- “这只票要起飞”
- “稳稳吃肉”
- “评论区发名单”

## 成稿匿名化规则

默认输出的“视频文案”必须匿名化处理，尤其是面向抖音口播时：

- 不写具体上市公司名、股票代码、简称、交易所、股价、涨跌、估值。
- 不把 2 家以上上市公司并排列出；并排列出会形成“名单感”。
- 不说“某龙头”“某标的”“某票”，这些仍然像股票语境。
- 优先用产业角色表达：上游高频材料厂、头部云厂商、海外 GPU 平台方、高端 PCB 制造商、光模块厂、设备平台方、封装测试厂、系统集成商、运营商客户。
- 如果必须说明证据来源，放在“素材来源/核验笔记”里，不放入口播正文；口播只说“公开披露材料里能看到”“行业会议资料显示”“标准组织文件里能查到”。
- 如果用户明确要求列公司，单独放到“内部核验表/非口播资料”，并再次提醒不能作为推荐、名单或排名。

推荐口播表达：

```text
不要说：深南电路、沪电股份、胜宏科技的年报显示……
改成：几家高端 PCB 厂商的公开披露材料里，已经能看到 AI 服务器、HPC 或数据通信相关产品的收入和订单变化。这里不是为了列公司，而是说明验证方法：看高端板占比、客户认证、批量交付和产品结构。
```

```text
不要说：某某是龙头，某某弹性更大。
改成：这个环节真正值得拆的不是谁名字更响，而是谁掌握高多层、低损耗材料、背钻、压合和良率控制这些关键能力。
```

## 固定免责声明

推荐使用：

> 本内容只做公开资料学习和产业链科普，不构成任何投资建议。评论区不讨论买卖点、不发代码、不建群。

使用要求：

- 免责声明要出现在视频结尾、置顶评论或简介中。
- 免责声明不能替代正文合规。
- 如果正文已经在暗示买卖、收益或名单，必须重写正文，不能只加免责声明。

## 公司案例合规边界

允许讲：

- 内部研究阶段可以看公司处在哪个产业链环节、相关业务、收入占比、客户认证、订单、毛利率和公开资料来源。
- 面向抖音的口播正文默认不出现具体公司名称，而是改写成“某类产业角色 + 可验证口径”。
- 证据截图或资料卡可以使用公开来源，但画面要突出“研究方法/资料来源”，不要突出股票名称、代码或公司列表。

禁止讲：

- 能不能买。
- 股价高不高。
- 是否低估。
- 是否要起飞。
- 目标价。
- 短期涨跌。
- 多家公司排名。
- 受益标的名单。
- 在口播中连续点名多家上市公司。
- 用公司名称承接“受益、兑现、订单、毛利、弹性、龙头”等词。

## 研报使用边界

允许：

- 把研报作为线索。
- 提炼行业问题。
- 查找术语和产业环节。
- 用公告和财报核验。
- 自己重构图表。

禁止：

- 搬运研报原文。
- 改写研报段落。
- 搬研报图表。
- 搬评级和目标价。
- 搬受益标的名单。
- 把研报结论包装成自己的观点。

## 发布前检查

发布前必须确认：

- 没有买卖建议。
- 没有股票池语义。
- 没有预测涨跌。
- 没有收益承诺。
- 没有私信、进群、评论区发代码。
- 公司只是案例，不是推荐结论。
- 研报没有被洗稿。
- 事实有公开来源支撑。
- 素材来源可说明，不侵犯明显版权。
