# 深度产业链分析方法

本文件用于所有产业链选题。目标是把热点拆成“产业判断”，不是写成概念解释、技术百科或公司名单。

## 0.0 旧账号式深度产业判断链

这个判断链是默认底稿，不是可选加分项。旧账号最值得迁移的能力，是把市场分歧、新闻刺激或研报结论，拆成一条可以被产业证据验证的逻辑链。每次分析一个产业，都先在内部写完这张卡，再决定口播怎么压缩。

```markdown
| 判断项 | 必须回答的问题 | 证据口径 | 反证条件 |
| --- | --- | --- | --- |
| 新闻/分歧入口 | 这件事为什么现在被讨论，是新闻、政策、财报、订单、价格、标准、客户路线还是供需状态变化？ | 事件原文、公告、财报、标准、客户材料、权威媒体 | 事件不能改变需求、供给、成本、路线或价格 |
| 市场浅层读法 | 普通讨论停在什么表层结论，是概念热、涨价、订单、国产替代、AI 映射还是公司名单？ | 社媒观点、研报摘要、新闻标题 | 只剩情绪，没有产业变量 |
| 真实产业问题 | 把浅层结论翻译成一个产业问题：谁卡供给、谁买单、哪条路线被迫切换、哪个环节价值量提升？ | 产业链资料、产品手册、招标、客户验证 | 问题不能被公开资料验证 |
| 行业状态 | 行业是放量、出清、价格战、库存去化、技术切换、认证重估还是商业化验证？ | 价格、库存、开工率、订单、资本开支、毛利率 | 状态判断和数据相互矛盾 |
| 旧瓶颈 | 旧路线/旧叙事为什么不够，是功耗、良率、材料、设备、成本、认证、交付还是系统集成卡住？ | 技术白皮书、产品参数、良率/成本/交付周期披露 | 旧路线没有明显边界，切换逻辑不成立 |
| 新路线 | 新路线具体改变哪个输入、工艺、结构、参数、成本或商业模式？处在样品、送样、小批量还是量产？ | 标准、产品发布、客户认证、设备订单、量产进度 | 停留在概念或样机，无法规模化 |
| 供应端 | 谁卡原料、产能、交付、扩产周期、库存、海外限制或供应纪律？ | 产能公告、交期、库存、进口替代、价格曲线 | 供给快速释放或库存过高 |
| 设备/工艺端 | 哪些设备、工艺窗口、良率、测试、封装、认证决定放量速度？ | 设备厂订单、工艺参数、良率、测试标准、客户验证 | 设备不稀缺、良率爬坡顺利、认证不构成门槛 |
| 上下游权力 | 谁压价、谁有定价权、谁能替代、谁会卷价格、谁掌握客户入口？ | 客户集中度、供应商数量、议价条款、替代路线 | 下游压价强、替代方案更便宜 |
| 价值量/利润池 | 钱从哪里流向哪里，谁拿高毛利，谁只是加工费，谁可能量大但不赚钱？ | BOM、ASP、收入占比、毛利率、单机价值量、成本结构 | 价值量提升不能转化为收入或利润 |
| 供需缺口 | 缺的是需求、产能、材料、良率、设备、认证还是交付周期？是真缺口还是情绪化抢货？ | 订单、排产、价格、库存、交期、招标、客户拉货节奏 | 价格不涨、库存上升、交期缩短、订单取消 |
| 验证和反证 | 至少 5 个行业专属验证口径，以及 3 个会推翻逻辑的信号。 | 本行业具体指标，不写泛泛的“看订单/看毛利” | 关键指标无法交叉印证 |
```

内部判断必须先过两道门：

- 深度门：不能只讲上中下游，必须讲清“旧瓶颈 -> 新路线 -> 供应/设备/工艺卡点 -> 价值迁移 -> 证据 -> 反证”。
- 合规门：所有“能不能做、拿住、机会、主线、增持、跑不跑、低位、补涨、弹性、目标价、受益标的”都改写成“产业逻辑是否成立、公开资料如何验证、反证条件是什么、行业角色是什么”。

## 0. 产业分析入手顺序

分析任何产业，先按“边界 -> 基本盘/增长盘 -> 状态/拐点 -> 结构 -> 路线 -> 卡点 -> 利润池 -> 成熟度 -> 证据”推进。不要一上来写上中下游，也不要一上来解释名词。

必须先完成 8 个判断：

- 行业边界：这个选题到底属于哪个产业，不属于哪个产业；终端应用、核心产品、关键材料、设备和服务分别是什么。
- 基本盘和增长盘：行业现有收入主要来自哪里，未来增量来自哪里；当前需求、增量需求、周期需求、政策需求和主题炒作需求不能混在一起。
- 行业状态和拐点：行业处在导入期、成长期、成熟期、库存去化、价格战、产能过剩、技术切换、政策驱动还是商业化验证期；是否真的出现拐点，拐点来自需求、价格、成本、技术、政策、供给、客户认证还是财报兑现。
- 竞争结构：上游供应商、下游客户、替代方案、新进入者、现有竞争者分别对利润有什么影响。不要直接套“五力”术语，口播里翻译成“谁压价、谁卡供给、谁能替代、谁会卷价格、谁有进入门槛”。
- 路线选择：技术路线、材料路线、工艺路线、商业路线和应用路线分别有哪些，行业当前更偏向哪条路，为什么。
- 关键卡点：真正限制产业放量的是效率、成本、寿命、良率、认证、安全、供给、标准、系统集成，还是商业模型。
- 利润池：收入大不等于利润厚。必须判断利润主要沉在哪个环节，谁有定价权，谁只是加工费，谁容易被客户压价或被价格战打穿。
- 成熟度证据：这件事处在论文、样品、送样、试点、小批量、规模量产、价格战中的哪一段；每一段需要的证据不同。

内部研究表必须先填这张，不必在口播里原样念：

```markdown
| 判断项 | 结论 | 证据来源 | 不确定性 |
| --- | --- | --- | --- |
| 行业边界 |  |  |  |
| 基本盘 |  |  |  |
| 增长盘 |  |  |  |
| 行业状态/拐点 |  |  |  |
| 竞争结构 |  |  |  |
| 主流路线 |  |  |  |
| 关键卡点 |  |  |  |
| 利润池 |  |  |  |
| 成熟度 |  |  |  |
```

如果这张表填不出来，说明还没有找到产业关键点，不能直接写成五分钟稿。

## 0.1 基本盘、增长盘和行业状态

每个行业必须先回答“原来靠什么赚钱，未来靠什么增长，现在处在什么状态”。这是从股票基本面研究中迁移来的行业分析方法，但用于短视频时必须去股票化、去荐股化。

必须拆清：

- 基本盘：行业现在真实的存量需求和收入来源。例如逆变器的基本盘可能来自光伏并网逆变器和存量替换，不能只写“新能源增长”。
- 增长盘：未来增量来自哪里。例如储能 PCS、构网型逆变器、海外并网标准升级、工商业储能、虚拟电厂、微电网等。
- 周期盘：是否受库存、价格战、项目招标节奏、补贴退坡、海外政策、汇率和渠道库存影响。
- 拐点：需求拐点、价格拐点、技术拐点、政策拐点、供给拐点、客户认证拐点、财报拐点分别是什么；没有证据就不能写成已经拐点。
- 当前状态：行业是高景气扩张、库存去化、价格战、技术切换、海外认证重估、政策约束加大，还是新技术小批量验证。

内部必须形成这张表：

```markdown
| 维度 | 具体内容 | 当前状态 | 验证指标 | 反证条件 |
| --- | --- | --- | --- | --- |
| 基本盘 |  |  |  |  |
| 增长盘 |  |  |  |  |
| 周期盘 |  |  |  |  |
| 行业拐点 |  |  |  |  |
| 当前格局 |  |  |  |  |
```

口播必须把这层判断自然讲出来。不要只说“新变量是某技术”，还要说清“旧基本盘是否还稳、未来增量来自哪里、行业现在是放量还是出清、价格战有没有结束、真正拐点能不能从订单、库存、毛利率、招标、认证和政策里看到”。

## 0.2 精选稿三道生产工序

精选稿不是最后靠自检拦出来的，而是成稿前必须按三道工序生产出来。每次写稿前必须先完成以下三步，再进入产业链拆解和口播写作。

### 0.2.1 行业基本盘和状态

先判断行业现在到底是什么状态，不要直接讲技术新变量。

必须回答：

- 基本盘：行业现在真实收入和需求主要来自哪里，老需求是稳定、下滑、修复还是价格战。
- 增长盘：未来增量来自哪里，是真实需求、政策驱动、技术切换、海外认证、存量替换，还是主题映射。
- 周期盘：库存、价格、招标、渠道、补贴、资本开支、海外政策是否正在影响行业。
- 当前格局：行业是放量、出清、价格战、库存去化、技术切换、政策驱动、认证重估，还是商业化验证。
- 拐点：有没有真正拐点；拐点来自需求、价格、成本、技术、政策、供给、客户认证、订单还是财报。

成稿必须自然写出这层判断。没有这层判断，文案只能算产品科普，不算精选产业稿。

### 0.2.2 精选级新闻由头

再找一个能支撑“为什么现在讲”的由头。开头不能只做定义、场景想象或概念纠偏。

可用由头包括：

- 政策/标准变化：监管、并网、出口、认证、安全、行业标准发生变化。
- 客户变化：大客户导入、招标、认证、采购口径、项目交付或供应链调整。
- 技术路线变化：旧路线遇到瓶颈，新路线从样品/试点进入验证或量产。
- 财报/订单变化：收入结构、毛利率、库存、资本开支、订单、产能出现变化。
- 行业状态变化：库存去化、价格战缓和、供需缺口、海外政策或安全审查改变采购逻辑。
- 巨头路线变化：平台方、头部客户、标准组织或龙头厂商公开改变路线。

如果找不到强新闻由头，也要明确写成“这期不是因为某个发布会，而是因为行业状态正在变”。不能假造新闻。

新闻由头不能虚化。禁止只写“公开资料显示”“公开资料里能看到”“有资料说”“行业都在讲”。必须写清楚来源类型和它证明了什么，例如：

```text
海外 GPU 平台方官网披露某整柜方案采用液冷架构，说明高功率整柜已经把散热从机房风道问题推到整柜系统设计问题。
```

口播正文可以不堆链接和公司名，但内部核验必须知道来源来自官网、标准、公告、客户招标、财报、产品手册、会议资料还是权威媒体。找不到来源时，只能写“待核验线索”，不能当作开头主线。

### 0.2.3 价值量和验证

最后才进入产业链拆解。拆解的重点不是“有哪些环节”，而是“钱往哪里流，怎么证明”。

必须回答：

- 价值量提升来自哪里：材料、设备、算法、认证、系统集成、售后网络、软件平台、标准壁垒、客户黏性，还是规模制造。
- 谁赚钱，谁赚辛苦钱：哪个环节有定价权，哪个环节被客户压价，哪个环节容易价格战。
- 谁先兑现：订单、客户认证、产能、收入占比、毛利率、库存、招标、标准、资本开支里，哪些已经能看到。
- 谁只是概念：只有发布会、样机、口号、概念映射，还是已经进入客户系统和公开披露材料。
- 反证是什么：什么数据恶化说明这条主线错了。

成稿必须至少给出 5 个本行业具体可查的验证口径。不能只写“看订单、看客户、看毛利率”；必须写清本行业应该看哪类订单、哪类客户、哪种认证、哪项收入占比、哪种库存/价格/招标/标准变化。

## 1. 先拆路线，再拆产业链

深度产业分析的第一步不是画上中下游，也不是解释概念，而是先拆“路线”。路线包括技术路线、材料路线、工艺路线、商业路线、应用路线和新旧替代路线。一个行业能不能找到关键，先看不同路线谁在解决同一个瓶颈，以及行业最终更可能选择哪条路线。

### 1.1 路线发现硬流程

路线不能只靠常识补全。每次遇到新产业或技术密集型产业，先联网做“路线发现”，再判断路线强弱。至少查 4 类来源：

- 头部客户/平台方路线图：看客户未来会采用什么路线、从哪一年开始验证、哪些指标被推高。
- 材料/设备/工艺厂商资料：看真正卡在材料、设备、加工窗口、良率、检测还是封装/集成。
- 标准组织、行业协会、权威产业媒体或学术综述：看路线命名、成熟度、产业共识和争议。
- 旧路线和替代路线资料：看旧路线是否还能升级，避免把“新路线出现”误写成“旧路线马上淘汰”。

搜索词必须同时覆盖中文和英文，不能只搜一个热词。默认组合：

```text
{选题} 技术路线 / 材料路线 / 工艺路线 / 标准 / 路线图 / 替代方案 / 良率 / 客户认证
{topic} roadmap / architecture / technology route / material / process / standard / alternative / limitation / customer adoption
```

路线发现必须形成内部表：

```markdown
| 路线类型 | 路线名称 | 它替代或补充谁 | 工作机制 | 当前阶段 | 采用方/验证方 | 最大卡点 | 价值量流向 | 反证条件 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 旧路线升级 |  |  |  |  |  |  |  |  |
| 新材料路线 |  |  |  |  |  |  |  |  |
| 新工艺路线 |  |  |  |  |  |  |  |  |
| 新架构/系统路线 |  |  |  |  |  |  |  |  |
| 替代路线/竞争路线 |  |  |  |  |  |  |  |  |
```

如果这张表只填出一行，不能直接成稿。必须继续查“旧路线升级”和“替代路线/竞争路线”。如果公开资料仍不足，正文要明确降级为“当前可确认主线”，不要写成全行业定论。

每个产业选题必须先完成路线对比：

```markdown
| 路线 | 工作原理 | 解决的问题 | 关键部件/工艺 | 优点 | 缺点 | 适用场景 | 成熟度 | 当前行业选择 | 关键壁垒 | 价值量流向 | 验证口径 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
```

路线分析必须回答：

- 这个行业到底有哪几条主流路线。
- 每条路线具体怎么工作：输入是什么、经过哪些核心部件/工艺、输出是什么、系统边界在哪里。
- 每条路线解决的是成本、效率、良率、寿命、密度、时延、功耗、可靠性、重量、认证还是规模化问题。
- 每条路线的短板是什么：成本高、材料稀缺、良率低、寿命短、设备贵、客户认证慢、标准不成熟，还是商业模型不成立。
- 每条路线适合什么场景：新建还是改造，高端还是低端，大客户还是长尾客户，高功率还是普通功率，短期可落地还是长期储备。
- 每条路线的系统代价是什么：改造难度、维护方式、责任边界、客户认证、供应链成熟度和失败风险。
- 目前行业选择哪条路线，为什么选它；如果还没有统一路线，要说明处于“路线竞争期”。
- 哪条路线最可能带来价值量提升，钱会流向材料、设备、制造、封装、系统集成还是运营环节。
- 哪些证据能验证路线：客户试用、样品、认证、小批量、量产、成本曲线、效率数据、寿命数据、订单、公开披露材料。

没有路线发现和路线分析，后面的上中下游容易变成名词堆砌。成稿前必须先判断：这是单一路线放量，还是多路线竞争；是旧路线升级，还是新路线替代；是材料决定胜负，还是设备/工艺/系统集成决定胜负。

口播里至少要把关键路线讲到“听得懂工作方式”的程度，不能只说路线名字。例如不能只写“冷板式更现实、浸没式更强、微通道更前沿”，必须讲清：冷板式是把冷板贴近高热芯片，液体在冷板内部流道带走热量；浸没式是把服务器或部件浸入绝缘冷却液，让热直接进入液体；微通道是把更细的流道做进冷板、封装或芯片附近，提高局部换热能力；喷淋式是把冷却液定向喷到高热表面或结构上，提高局部换热但带来液体管理和可靠性要求。其他行业也一样，路线名字必须落到工作机制和产业代价。

## 2. 再拆口径

每个选题先拆 5 个口径：

- 大行业口径：属于 AI 算力、半导体、光通信、新能源、机器人、消费电子、工业软件等哪类大行业。
- 细分产品口径：是产品、材料、设备、零部件、工艺、封装模块、系统集成，还是下游应用。
- 自身产业链口径：上游材料/设备/耗材/零部件，中游制造/封装/集成/测试，下游客户/应用分别是什么。
- 下游应用口径：放到 AI 服务器、数据中心、新能源车、通信网络、工厂自动化等更大链条里处在什么位置。
- 市场预期口径：外部真正关注的是需求放量、国产替代、涨价、供需缺口、路线切换、客户认证，还是概念映射。

如果一个词有多个口径，必须先拆开。例如“光芯片”不能直接等同于所有光通信芯片，至少要区分激光器、调制器、探测器、硅光 PIC、CW 光源、EML、VCSEL、DFB，以及 TIA/DSP 等电芯片或模拟/数字处理环节。

口径拆分还必须避免“终端热度替代核心产业”。例如太空光伏不能只按“发电、传电、收电、并网”写模块，因为那只是系统流程；真正的产业入手点要先拆空间太阳能电池路线、无线传能路线、展开结构路线和在轨装配路线，再判断哪条路线进入工程验证。

## 3. 再拆行业格局

所有产业都必须讲上中下游格局，不能只讲技术点：

- 上游：材料、设备、耗材、核心零部件、关键工艺；判断谁供给慢、谁有定价权、谁卡扩产。
- 中游：制造、封装、集成、测试、良率、规模化能力；判断谁是头部、谁追赶、谁容易价格战。
- 下游：客户、需求来源、付费方、导入周期、认证和替换成本。
- 价值量：钱主要被哪个环节赚走，哪个环节只是加工费，哪个环节毛利率可能更高。
- 竞争格局：海外头部、国内头部、二线追赶者、替代难度、价格战风险、客户集中度。
- 供需缺口：缺的是材料、设备、产能、良率、客户认证、标准成熟度，还是终端需求本身。
- 原材料稀缺性：关键原料是否稀缺，供给是否集中，价格是否波动，是否受出口管制、资源禀赋、提纯能力或上游产能限制。
- 卡脖子环节：真正卡住产业放量的是材料、设备、工艺、良率、设计、认证、系统集成、成本，还是商业模式。

内部必须先形成判断表，视频正文可以自然口播，不必念表：

```markdown
| 环节 | 做什么 | 价值量/利润池 | 头部厂商/代表公司 | 壁垒 | 供需状态 | 验证口径 |
| --- | --- | --- | --- | --- | --- | --- |
| 上游材料/设备 |  |  |  |  |  |  |
| 核心零部件/芯片/基材 |  |  |  |  |  |  |
| 制造/封装/集成/测试 |  |  |  |  |  |  |
| 下游客户/应用 |  |  |  |  |  |  |
```

价值量/成本/利润池有可靠来源时给具体值；没有来源时只能写“暂无可靠占比，需核验”或“估算区间，需核验”，不能编造精确百分比。

## 4. 必须拆关键卡点

路线之后必须拆关键卡点。不要平均罗列每个环节，要判断哪一个环节最可能决定产业能否放量。

```markdown
| 卡点 | 具体表现 | 卡在谁手里 | 壁垒来源 | 是否稀缺 | 价值量/利润池 | 验证指标 | 反证条件 |
| --- | --- | --- | --- | --- | --- | --- |
```

卡点分析必须回答：

- 技术壁垒：效率、良率、寿命、精度、可靠性、热管理、系统控制、规模化一致性。
- 原材料稀缺性：是否依赖稀有金属、高纯材料、特殊设备、特种气体、空间级材料或进口部件。
- 设备和工艺壁垒：核心设备是否难买，工艺窗口是否窄，良率是否难提升。
- 客户认证壁垒：客户为什么不轻易换供应商，认证周期多长，失败成本多高。
- 价值量归属：利润主要被材料、设备、核心部件、制造、系统集成、运营服务中的谁拿走。
- 反证条件：如果这个卡点被快速突破或被替代，原来的产业逻辑是否失效。

关键卡点必须落到“谁卡住谁”。例如不是写“太空光伏难在效率和成本”，而是写清楚：电池路线卡在比功率、抗辐照、寿命和空间级封装；无线传能卡在远距离效率、波束控制和安全标准；工程系统卡在大型展开结构、热控、发射成本和在轨装配。

## 5. 必须判断价值量和利润池

产业分析不能只说“有前景”，必须判断钱在哪里。

价值量必须拆成三层：

- 成本价值量：谁在 BOM、制造成本、设备折旧或系统成本里占钱多。
- 利润价值量：谁有毛利率、定价权、认证壁垒、客户黏性和替代难度。
- 瓶颈价值量：谁决定产品能不能做出来、良率能不能起来、客户认证能不能过、行业能不能放量。

不能把成本占比高直接写成高价值环节。大宗材料可能成本占比高，但利润和瓶颈不一定高；低占比添加剂、表面处理、检测设备、关键制程或客户认证经验，可能成本占比不高，却卡住良率和放量。

必须回答：

- 产业总成本里，哪几个环节价值量最高。
- 哪些环节成本价值量高但利润低。
- 哪些环节成本价值量不高但利润价值量或瓶颈价值量高。
- 哪个环节毛利率可能最高，为什么。
- 哪个环节只是加工费或规模制造，容易价格战。
- 哪个环节扩产最慢，最可能形成供需缺口。
- 哪个环节有定价权，哪个环节被客户压价。
- 如果没有公开百分比，至少给“高/中/低 + 原因 + 待核验口径”，不能凭空编精确数据。

内部必须形成价值量判断表：

```markdown
| 环节 | 成本价值量 | 利润价值量 | 瓶颈价值量 | 定价权 | 扩产难度 | 价格战风险 | 验证口径 |
| --- | --- | --- | --- | --- | --- | --- | --- |
```

利润池判断要和竞争结构连在一起：如果上游材料稀缺、认证周期长、替代难，利润可能向上游集中；如果下游客户集中且压价强，中游制造即使收入增长，也可能只是赚辛苦钱；如果标准和生态由平台方控制，利润可能不在硬件制造，而在平台、软件、认证或服务。

材料、设备、技术/工艺三端必须分开判断：

```markdown
| 端口 | 成本价值量 | 利润价值量 | 瓶颈价值量 | 产业判断 |
| --- | --- | --- | --- | --- |
| 材料端 | 是大宗原料、专用材料、高纯材料、添加剂还是耗材 | 是否有配方、纯度、一致性、认证或供应稀缺 | 是否卡性能、寿命、良率、交付或国产替代 | 普通原料 / 关键材料 / 卡脖子材料 |
| 设备端 | 设备投资、折旧、维护耗材、单线资本开支 | 设备厂毛利、进口依赖、服务绑定、替代难度 | 是否卡扩产、精度、良率、交期和爬坡 | 普通设备 / 关键设备 / 扩产瓶颈 |
| 技术/工艺端 | 通常体现为良率损耗、制程窗口、测试认证和隐性成本 | 产品溢价、良率优势、客户黏性、制造 know-how | 是否决定性能、可靠性、量产稳定性和客户导入 | 可复制工艺 / 经验壁垒 / 核心 know-how |
```

判断示例：

- 铜箔里，铜原料可能是成本大头，但偏大宗属性；表面处理设备、添加剂体系、电沉积工艺、厚度控制、粗糙度和良率更可能决定高端铜箔的利润价值量和瓶颈价值量。
- EML 芯片里，磷化铟衬底/外延材料、半导体设备、芯片设计和制造技术都可能重要，必须分别判断材料质量、设备能力、外延/制程/良率和客户认证，不能简单归因到单一环节。

口播里不要说“根据五力模型、利润池模型”。要翻译成人话：

```text
这条链不是谁出货多谁就一定赚钱。要看谁卡供给、谁被客户压价、谁能被替代、谁扩产最慢，以及最后利润沉在哪个环节。
```

## 5.1 必须判断技术成熟度

新技术选题必须判断成熟度。不要把论文、样机、地面验证、客户试点和商业量产写成同一个阶段。

内部按这张表判断：

```markdown
| 阶段 | 典型证据 | 口播写法 | 不能写成 |
| --- | --- | --- | --- |
| 论文/实验室 | 论文、专利、实验数据 | 还在验证原理 | 已经产业化 |
| 样品/送样 | 样机、demo、送样公告 | 开始进入工程验证 | 已经放量 |
| 客户试点 | 客户测试、场景试运行 | 开始验证可用性和 ROI | 已经大规模采购 |
| 小批量 | 批量交付、产线验证 | 初步兑现，但规模和毛利待看 | 全行业爆发 |
| 规模量产 | 收入占比、产能、订单、毛利率 | 进入财报验证 | 只靠概念热度 |
| 价格战 | 扩产过快、毛利下滑、库存上升 | 从稀缺转向竞争 | 继续无脑高景气 |
```

太空光伏、固态电池、可控核聚变、量子计算、空芯光纤、CPO 等选题尤其要写清成熟度。不能因为有官方验证或巨头发布，就直接写成商业化拐点。

## 6. 必须拆路线分化

只要行业存在多技术路线、多材料路线、多设备路线、多封装路线、多应用路线或新旧路线替代，必须拆“路线分化”：

```markdown
| 路线/方案 | 解决的问题 | 关键环节 | 方案提供方 | 当前阶段 | 验证指标 | 反证条件 |
| --- | --- | --- | --- | --- | --- | --- |
```

路线分化必须回答：

- 它解决的是降本、提速、降功耗、提良率、增密度、降时延、提升可靠性、扩大产能，还是通过客户认证。
- 当前处在概念、样品、送样、验证、小批量、量产、放量、价格战中的哪一段。
- 钱会流向材料、设备、制造、封装、系统集成，还是下游客户。
- 哪些证据能验证：客户认证、订单、产能、收入、毛利率、库存、资本开支、良率。

## 7. 深扒颗粒度

所有产业都要下钻到本行业真正决定壁垒和价值迁移的颗粒度。深扒不是堆术语，每个专业词进入正文前，都要回答“解决哪个瓶颈、为什么现在价值提升、谁买单、怎么验证、最大反证是什么”。

示例颗粒度：

- 光芯片/光模块/CPO：400G/800G/1.6T、DFB、EML、VCSEL、CW 光源、硅光 PIC、薄膜铌酸锂、探测器、TIA、DSP、光引擎、可插拔、NPO、CPO、LPO、封装测试和良率。
- 光纤/光通信：G.652.D、G.654.E、G.657、IEC A1/OM、多模/单模、空芯、多芯/少模、连接器、微缆、数据中心布线和运营商集采。
- PCB/CCL：高多层、HDI、SLP、背钻、低 Dk/Df、PTFE/PPO/碳氢树脂、玻纤布规格、HVLP/VLP 铜箔、良率、认证和 AI 服务器单机价值量。遇到 AI 服务器 PCB、GPU 服务器 PCB、高速板、高多层板、CCL、电子布、铜箔等选题，不能只写“AI 带动 PCB 需求”“高端板更紧缺”。必须拆到以下最低颗粒度：
  - 下游平台和场景：AI 服务器主板、加速卡、交换机、背板、整柜方案和高速互联架构，说明是 GPU/CPU/HBM/交换芯片之间的带宽、功耗、信号完整性和板层空间压力在推高 PCB 要求。
  - 旧路线和旧瓶颈：普通 FR-4、传统 12-16 层板、普通减成法线路、常规 VLP 铜箔和普通玻纤布在哪些速率、层数、线宽线距、损耗、阻抗、压合对位和良率上不够用。
  - 材料路线：低损耗 CCL 必须拆树脂、铜箔、玻纤布三件事。树脂要讲 Dk/Df、PPO、碳氢树脂、PTFE、M7/M8/M9/M10 等级和客户认证；铜箔要讲 VLP/HVLP、表面粗糙度、趋肤效应、代际差异和供应集中度；玻纤布要讲低介电玻纤布、开纤布、石英布、尺寸稳定性和成本/工艺约束。
  - 制造路线：必须拆高多层压合、HDI、mSAP、背钻、激光钻孔、LDI 曝光、电镀、阻抗控制、AOI/电测和可靠性测试。mSAP 必须讲清和传统减成法的工艺差异、线宽线距提升、设备投资、设备交期和良率爬坡。
  - 设备/工艺卡点：至少覆盖激光钻孔机、LDI 曝光机、电镀线、压合设备、背钻设备、检测设备中的 3 类，并说明设备交期、产线投资、良率和客户认证如何限制扩产速度。
  - 价值迁移：必须判断钱是流向低损耗 CCL、HVLP 铜箔、低介电/石英布、高多层 PCB 制造、mSAP 产线、检测设备，还是下游服务器平台；不能平均说“PCB 都受益”。
  - 验证口径：至少给出 5 个 PCB 专属指标，包括 AI 服务器/数据中心 PCB 收入占比、高速材料收入占比、M7-M9/M10 认证进度、HVLP 铜箔代际和供应、mSAP/高多层产能利用率、设备订单和交期、良率、客户认证、涨价/降价传导、库存和应收账款。
  - 反证条件：必须写出 3 个以上，包括 GPU/云厂商资本开支下调、高端 CCL/PCB 扩产过快、设备交期缩短、客户认证延后、材料价格无法传导、封装/光互联/系统架构替代部分 PCB 价值量。
  - 数字约束：所有增长率、CAGR、毛利率、市场份额、设备投资额、交期、线宽线距、层数、速率和用量倍数必须有来源等级；不能把研报数字当事实直接口播。找不到来源时只能写“待核验口径”或改成方向性判断。
- HBM/先进封装：HBM3E/HBM4、TSV、微凸点、混合键合、中介层、CoWoS-S/R/L、TCB、underfill、ABF、玻璃 core substrate、玻璃 interposer、TGV、RDL、孔壁金属化、面板级加工、检测量测、良率和扩产周期。遇到玻璃基板这类选题，不能只写“玻璃替代 ABF”，必须拆旧路线 ABF 升级、新路线玻璃核心基板、玻璃中介层/转接结构、面板级封装路线，以及它们分别解决大尺寸、低损耗、尺寸稳定、布线密度、成本和良率中的哪一个问题。
- 液冷/电力：冷板式液冷、浸没式液冷、微通道液冷、喷淋式液冷、CDU、泵阀、管路、快接头、歧管、冷却液、材料兼容、漏液检测、PUE、单柜功率密度、配电、UPS、变压器和运维责任。必须讲清每条液冷路线怎么把热从芯片/板卡/整机带走，以及改造难度、维护责任、可靠性和客户认证差异。
- 储能：电芯体系、PCS、BMS、EMS、温控、消防、循环寿命、度电成本、电价机制、并网消纳、项目现金流和海外渠道。
- 半导体设备/材料：具体工艺步骤、设备类型、耗材属性、特气、光刻胶、靶材、抛光液、掩膜版、客户验证周期、良率贡献和国产替代难度。
- 机器人/物理 AI：仿真、端侧算力、感知、控制器、伺服驱动、电机、减速器、丝杠、编码器、灵巧手、关节模组、系统集成和单场景 ROI。
- 太空光伏/空间太阳能：先拆电池路线，例如 HJT、砷化镓/III-V 多结电池、钙钛矿/钙钛矿叠层等；再拆无线传能路线，例如微波、激光；再拆展开结构、热控、在轨装配、地面接收阵列和并网。不能只讲“发电、传电、收电、并网”模块。

太空光伏示例分析：

- HJT 路线：优势是产业基础和成本潜力，问题是空间环境下重量、效率、辐照稳定性和封装可靠性要验证。
- 砷化镓/III-V 多结路线：优势是空间应用成熟、效率高、抗辐照能力强，问题是成本高、材料和外延制造壁垒高，价值量可能集中在外延、芯片制造和空间级封装。
- 钙钛矿/叠层路线：优势是轻量、柔性、高比功率潜力，问题是寿命、湿热/辐照稳定性、面积放大和空间长期可靠性仍是验证核心。
- 不能只问“哪种电池效率高”，要问行业为什么选这条路线：比功率、寿命、辐照稳定性、成本、可展开面积、制造良率和在轨维护难度。
- 太空光伏的卡点不只在电池，还在无线传能效率、波束控制、热控、大型展开结构、发射成本、在轨组装、频谱与安全标准。必须判断哪一个卡点最先进入工程验证，哪一个卡点决定商业化。

## 8. 文案压缩规则

共识型瓶颈要一句话压缩，不要写成废话段。比如：

```text
高速互联的核心矛盾，是电信号路径越长，损耗、功耗和散热压力越大。
```

压缩后必须立刻进入产业判断：因此谁受益、谁被替代、谁能验证、谁只是概念。

每一段口播都要有一个产业落点：行业位置、上下游格局、价值量、竞争格局、客户验证、财报验证或反证风险。只有技术解释、比喻和顺滑表达，不算合格。

## 9. 成稿硬门槛

五分钟精选稿必须默认 1300 到 1700 个中文字符，并包含：

- 1 个新闻由头或产业变量。
- 1 段行业基本盘、未来增长盘、当前状态和拐点判断，说明这个行业现在是放量、出清、价格战、库存去化、技术切换、政策驱动还是商业化验证。
- 1 个行业边界和竞争结构判断，说明这条链里谁卡供给、谁压价格、谁能替代、谁有进入门槛。
- 1 段路线对比，说明行业当前在选哪条路线、为什么。
- 1 个旧瓶颈，但不能铺成废话。
- 3 到 4 个关键环节判断，每个都要讲“是什么、处在产业链什么位置、上中下游格局如何、为什么价值提升、价值量由谁拿走、怎么验证、最大反证是什么”。
- 2 到 3 个公司/平台案例，只作公开资料案例，不作投资结论。
- 至少 5 个具体验证口径，必须覆盖订单/客户/认证/产能/库存/收入占比/毛利率/资本开支/招标/标准变化中的多项。
- 1 段反证风险。
- 1 段研究方法收束。

当用户要求写稿、改稿、精选向文案或判断能不能进精选时，默认直接输出修好的完整成稿。不要只给评分和建议。

## 10. 精选稿出厂闸门

成稿前必须用“精选眼光”卡一次。普通科普只要讲清楚是什么，精选稿必须讲清楚“为什么现在值得讲、行业现在处在什么状态、钱为什么重新分配、证据能不能支撑”。

任何稿子出现以下问题，必须判定为 70 分以下，不能作为最终稿：

- 只有产品升级，没有行业基本盘、增长盘和当前状态判断。
- 只有概念纠偏，没有新闻由头、政策变化、客户变化、技术路线变化或财报/订单变化。
- 只有“某技术更重要”，没有讲旧市场是不是还稳、未来增量来自哪里。
- 只说“有溢价”，没有拆清溢价来自算法、材料、设备、认证、系统责任、售后网络、软件平台还是客户黏性。
- 只说“看订单、毛利率、客户验证”，没有写到本行业具体应该看哪些订单、哪类客户、哪种认证、哪项收入占比、哪种库存或价格变化。
- 只讲风险清楚，没有指出最可能打破逻辑的反证指标。

内部必须先完成这张精选闸门表：

```markdown
| 闸门 | 必须回答 | 不通过表现 | 本稿处理 |
| --- | --- | --- | --- |
| 为什么现在讲 | 最近变量、新闻由头、政策/客户/技术/财报变化 | 开头只是定义或概念纠偏 |  |
| 行业状态 | 基本盘、增长盘、周期盘、当前格局、拐点 | 只讲新技术，不讲行业处境 |  |
| 价值迁移 | 钱从哪里来，流向哪个高壁垒环节 | 只说“价值提升” |  |
| 关键环节 | 哪些环节真有壁垒，哪些只是执行 | 平均罗列上中下游 |  |
| 证据密度 | 订单、客户、认证、产能、库存、收入、毛利率、招标、标准 | 验证口径泛泛而谈 |  |
| 反证风险 | 哪个指标恶化说明逻辑错了 | 只写宏观风险 |  |
```

只有这张表能填实，才允许输出精选稿。填不实就继续联网查证或重写主线。
