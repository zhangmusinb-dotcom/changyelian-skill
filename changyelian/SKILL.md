# changyelian

## 核心目标

为“被套但体面”生产中文产业链科普内容。内容只做公开资料学习和产业链拆解，不做荐股、不做股票池、不做买卖建议。

默认目标是：**抖音精选向硬核产业链稿**。安全、顺滑、没错误的概述稿不算合格。成稿必须回答：

- 为什么现在讲这个选题。
- 行业现在处在什么阶段。
- 旧路线或旧瓶颈被什么变量推到极限。
- 新路线、新材料、新工艺、新设备或新架构解决了什么问题。
- 钱可能从哪里流向哪里。
- 哪些公开证据能验证。
- 哪些信号会推翻这条逻辑。

优先级：

```text
先深度 -> 再证据 -> 再合规 -> 最后口播自然
```

合规不能成为写浅的理由。先把产业研究做硬，再做匿名化和风险降级。

## 强制流程

不要直接写文案。每次按这个顺序推进：

1. 判断选题类型和产业边界。
2. 用公开资料核验事实。S/A 级来源优先，B 级只作线索，C 级不能当事实。
3. 先完成三道生产工序：
   - 行业基本盘和状态
   - 精选级新闻由头
   - 价值量和验证：必须拆成本价值量、利润价值量、瓶颈价值量，不能把成本占比高直接写成高价值环节
4. 成稿前完成路线发现：
   - 旧路线升级
   - 新材料路线
   - 新工艺路线
   - 新架构或系统路线
   - 替代路线或竞争路线
5. 完成 12 步产业判断底稿：
   - 新闻/分歧入口
   - 市场浅层理解
   - 真实产业问题
   - 行业状态
   - 旧瓶颈
   - 新路线
   - 供应端
   - 设备/工艺端
   - 上下游权力
   - 价值量/利润池
   - 供需缺口
   - 验证口径和反证条件
6. 涉及公司时，口播正文默认改成产业角色；具体公司证据放在核验笔记，不放进口播正文。
7. 写精选向短视频时，搜索并学习抖音或 YouTube 公开视频。公开视频只学表达和选题线索，不能当事实依据。
8. 输出文案、分镜、素材清单、标题封面建议和合规检查。
9. 如果已保存成稿文件，运行质量脚本：

```text
python scripts/draft_quality_check.py <draft.md>
```

脚本不通过，先改稿再交付。

## 硬闸门

### 证据

用户要深度分析、精选稿、五分钟稿或改稿时，不要把关键证据甩给用户补。能查就查；查不到就说明已查哪些口径、哪些只能降级。

所有数字必须有来源等级。行业增速、CAGR、市场份额、毛利率、资本开支、设备交期、层数、线宽线距、信号速率、用量倍数等，不能无来源写成确定事实。没有来源时，写成“待核验口径”或改成方向性判断。

### 路线发现

没有路线发现表，不进入最终成稿。

路线表至少要包含：旧路线升级、新路线、替代/竞争路线，以及工作机制、当前阶段、最大卡点、价值量流向和反证条件。

### 普通科普稿淘汰

如果稿子只是说“行业变重要了、需求增长了、AI 带动了”，直接判定为普通科普稿。

缺少以下任意关键项，也不能交付为精选稿：

- 材料等级或技术标准
- 工艺路线
- 设备或产能约束
- 成本价值量、利润价值量和瓶颈价值量的区分
- 良率、认证或客户验证
- 本行业专属验证口径
- 反证信号

### 合规

禁止输出：

- 荐股、股票池、买卖建议、目标价、涨跌预测、收益承诺
- 私信、进群、评论区发代码等引流
- 核心标的、最受益、龙头股、低位潜伏等交易化表达
- 口播正文里的上市公司名单、股票代码、排名和并列推荐

口播正文默认使用产业角色，不直接点名上市公司。

### 风格

写得像一个视频博主在讲自己的研究，不像 PPT、百科、研报复读或 AI 提纲。

避免：

- “第一层/第二层/第三层”
- “第一块/第二块/第三块”
- “首先/其次/最后/总而言之”
- 连续“不是……而是……”
- 连续反问推动段落
- 公司名单感和筛股票口吻

优先用正向产业判断推进，不要靠否定纠偏凑节奏。

## 产业分析方法

主文件不堆行业例子。任何行业都先找到本行业最小有效颗粒度：

```text
标准 / 型号 / 材料等级 / 工艺路线 / 设备 / 客户场景 / 验证口径
```

再按这条链写：

```text
旧路线边界 -> 新路线变化 -> 谁卡瓶颈 -> 价值迁移 -> 怎么验证 -> 什么会证伪
```

价值迁移必须分三层判断：

```text
成本价值量：谁在 BOM、制造成本、设备折旧或系统成本里占钱多
利润价值量：谁有毛利率、定价权、认证壁垒、客户黏性和替代难度
瓶颈价值量：谁决定良率、扩产、交期、客户认证和产业能不能放量
```

材料、设备、技术/工艺要分开看。材料要判断是大宗原料、关键材料还是卡脖子材料；设备要判断是普通产线设备、关键设备还是扩产/良率瓶颈；技术/工艺要判断是可复制经验，还是决定性能、可靠性、量产稳定性和客户导入的核心 know-how。

比如 PCB/CCL 题材，最低检查不是“PCB 需求增长”，而是材料等级、工艺路线、设备交期、良率、客户认证和反证指标。具体颗粒度放在 [references/deep_industry_method.md](references/deep_industry_method.md)，必要时再看 `examples/` 校准结构。

## 五分钟稿标准

默认 1300-1700 个中文字符。

必须包含：

- 1 个可核验新闻由头或产业变量
- 行业基本盘、增长盘和当前状态
- 旧瓶颈
- 路线对比
- 3-4 个关键环节判断
- 至少 5 个本行业具体验证口径
- 至少 2 个反证风险
- 回到研究方法的结尾
